# 2.4 路由与页签模型

系统采用：

```text
菜单 -> 路由 -> 页面 -> 页签
```

的统一模型。

## 路由规范

路由路径应体现业务层级：

```text
/order-manage/sale-order
/booking/sea-booking
/finance/receivable
```

路由 meta 至少包含：

* title：页面标题
* menuKey：对应菜单 key
* module：所属模块
* keepAlive：是否缓存
* permission：权限标识

## 页签规范

顶部 tabnav 表示已打开页面，不是主菜单。

页签必须遵循：

* 只显示真实打开页面
* 当前页签必须高亮
* 页签标题来自路由 meta
* 首页或固定页签不可关闭
* 禁止用不存在的页签填充视觉空间

## 验收标准

* 点击菜单是否能打开对应路由？
* 当前路由是否能反向高亮菜单？
* 当前路由是否能生成页签？
* 页签是否只显示真实页面？
