# 2.3 菜单信息架构

系统采用三层菜单模型：

```text
一级模块
  二级业务域
    三级功能页
```

示例：

```text
订单管理
  进出口订单
    业务单
    所有订单
  出口订单
    出口海运订单
    出口空运订单
    出口铁路订单
```

## 菜单设计要求

* 一级模块必须按业务域组织，不按技术模块组织。
* 菜单名称必须使用业务人员理解的行业语言。
* 当前页面必须有明确高亮。
* 不存在的菜单不得为了视觉完整而虚构。
* 大项目中菜单配置必须数据化，禁止硬编码在页面组件中。
* 菜单配置必须与路由、权限、页签保持同一数据源或可追溯映射。

## 大项目菜单扩展规则

菜单配置建议至少包含：

```text
key
title
routeName
permission
icon
children
order
hidden
```

## 禁止项

* 禁止在页面组件里写死菜单。
* 禁止菜单与路由没有映射关系。
* 禁止为了视觉效果虚构菜单。
