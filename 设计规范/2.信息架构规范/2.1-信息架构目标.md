# 2.1 信息架构目标

国际货代系统的信息架构目标不是“把菜单列出来”，而是让复杂业务在大规模菜单、大量字段、多角色协作下仍然保持清晰、可预测、可扩展。

系统必须支持：

* 几百个菜单的长期扩展
* 多业务模块并行
* 多角色权限差异
* 高频列表页操作
* 多状态、多节点、多单据关系

信息架构设计必须优先保证用户能够快速回答：

```text
我在哪个业务模块？
我当前在处理哪类单据？
我现在能做什么？
哪些信息最重要？
```

## 设计原则

* 业务域优先于技术模块
* 主业务链路优先于辅助功能
* 高频入口优先于低频入口
* 真实菜单优先于视觉填充
* 可扩展配置优先于页面硬编码

## 验收标准

* 菜单是否按业务域组织？
* 页面是否能明确表达当前业务位置？
* 信息层级是否清楚？
* 是否适合几百菜单规模继续扩展？
