# 海运出口订单 UI 成熟度验收

日期：2026-07-22  
范围：应用 Shell、海运出口订单工作台、快速查看抽屉、完整详情、业务 Modal  
基线：Vue 3 + Arco Design Vue + GI theme + VXE Table

## 主题所有权

- 唯一 Arco CSS baseline / palette：`@arco-themes/vue-gi-demo/css/arco.css`
- `global.css` 只负责语义 alias、密度、布局 token、状态语义和 VXE bridge
- 无默认 Arco CSS 重复导入，无 `theme.css` 第二套主题适配层
- 页面未重写 GI 主色、组件皮肤或 popup 视觉
- 侧栏只呈现已注册路由；静态业务域标题 + 单一入口选中，避免父子双高亮和不可用模块造成模板化 Admin 感

## 交互证据

- 查询方案：系统方案应用、个人方案创建、重命名、更新、复制、设为默认、重置回默认、删除均可执行。
- 查询方案数据有 `version/revision/owner/default/updatedAt`，支持 v1 到 v2 本地迁移和同名冲突拦截。
- 团队共享方案需要服务端共享存储与权限，当前不在正常 UI 暴露，避免用 localStorage 冒充共享。
- 主表：关键列与操作列固定，横向滚动归 VXE 所有；密度 36/44；列设置与分页位于数据面。
- 批量部分失败：成功数、失败数、失败订单号在表格原位显示，失败行保持选中。
- 快速查看：720px 标准抽屉保留列表上下文，完整详情作为复杂编辑入口。
- 费用、节点、异常明细默认展示文本；仅当前编辑行显示控件；保存/取消不改变列宽和表格结构。
- 费用空名称、非正金额等错误同时标记单元格并在所属表格上方汇总；失败输入不会丢失。
- 行编辑期间其他详情页签禁用；离开路由需要确认，避免未保存修改被隐藏或丢失。
- 状态修改只展示当前状态允许的下一状态；原因和目标状态使用 Arco Form 原位校验。

## 确定性边界态

QA 可通过 `uiState` 查询参数复现，不在正常 UI 暴露开发开关：

- `loading`, `slow`, `empty`, `error`, `permission`
- `partial`（批量部分失败）
- `long`（超长客户、航线和异常文本）
- `extreme`（极端柜量和日期）

错误态提供重试；权限态隐藏写操作；慢请求保留 loading；部分失败保留失败选择。

## 能力边界

- 共享查询方案需要服务端共享存储与权限接口后才可开放。
- 文件上传需要项目共享上传服务后才可开放；当前入口明确禁用，不使用 `a-upload` 绕过契约。
- 当前构建仍有 Vite 大 chunk 警告，不影响本次 UI/交互验收，但应在后续性能专项处理依赖拆包。
